# 电厂巡检机器人视频传输系统 - 快速启动指南

## 📋 系统要求

### 硬件要求
- **摄像头**: USB摄像头或内置摄像头
- **内存**: 至少 4GB RAM
- **网络**: 局域网环境，建议 10Mbps+ 带宽

### 软件要求
- **Python**: 3.9+ (推荐 3.11+)
- **Node.js**: 18+ (推荐 20+)
- **FFmpeg**: 4.4+ (用于视频编码)
- **浏览器**: Chrome 90+, Firefox 88+, Safari 14+

npm要换源

## ⚠️ 重要前置条件

### 🔴 必须安装（否则无法运行）

#### 1. 安装FFmpeg

**摄像头采集端需要FFmpeg进行视频编码！**

```bash
# macOS
brew install ffmpeg

# Ubuntu/Debian
sudo apt install ffmpeg

# 验证安装
ffmpeg -version
```

#### 2. 安装并启动MediaMTX（RTSP服务器）

**🚨 这是关键步骤！没有MediaMTX，视频流无法工作！**

MediaMTX是一个轻量级的RTSP/RTMP/HLS媒体服务器，用于接收摄像头推送的视频流。

```bash
# macOS (使用Homebrew) - 推荐
brew install mediamtx

# 启动MediaMTX（保持此终端运行）
mediamtx
```

**或者下载二进制文件：**
```bash
# 访问 https://github.com/bluenviron/mediamtx/releases
# 下载 mediamtx_vX.X.X_darwin_amd64.tar.gz (macOS Intel)
# 或 mediamtx_vX.X.X_darwin_arm64.tar.gz (macOS Apple Silicon)

# 解压并运行
tar -xzf mediamtx_*.tar.gz
./mediamtx
```

**MediaMTX启动后会显示：**
```
2024/03/07 10:45:00 INF MediaMTX v1.x.x
2024/03/07 10:45:00 INF [RTSP] listener opened on :8554 (TCP), :8000 (UDP/RTP), :8001 (UDP/RTCP)
```

看到这个输出说明MediaMTX已成功启动！

---

### ⚠️ 如果不安装MediaMTX会怎样？

- ❌ 摄像头客户端会报错：`[Errno 32] Broken pipe`
- ✅ 设备仍会注册到服务器（可以在Web界面看到设备）
- ❌ **无法看到实时视频流**
- ❌ 无法录像和回放

**结论：要完整体验系统，必须安装MediaMTX！**

## 🚀 快速启动（3步）


### 第-1步：启动项目文档（保持此终端运行）

cd study

npm i

npm run dev


### 第0步：# 启动MediaMTX（保持此终端运行）
mediamtx

### 第一步：启动服务器

```bash
# 进入服务器目录
cd server

# 激活虚拟环境
source venv/bin/activate

# 初始化数据库（首次运行）
python -c "from models import init_db; import asyncio; asyncio.run(init_db())"

# 启动服务器
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

一行
cd server && source venv/bin/activate && uvicorn main:app --host 0.0.0.0 --port 8000 --reload

服务器将在 `http://localhost:8000` 启动

### 第二步：启动Web客户端

```bash
# 新开一个终端，进入Web客户端目录
cd web-client

# 启动开发服务器
npm run dev
```

一行
cd web-client && npm run dev

Web界面将在 `http://localhost:5173` 启动（如果端口被占用，会自动使用其他端口如5174）

### 第三步：启动摄像头采集端

```bash
# 新开一个终端，进入摄像头客户端目录
cd camera-client

# 激活虚拟环境
source venv/bin/activate

# 启动摄像头采集
python main.py
```

一行
cd camera-client && source venv/bin/activate && python main.py

## 📖 详细安装步骤

### 1. 安装系统依赖

#### Ubuntu/Debian
```bash

```

#### Windows
1. 下载并安装 [Python 3.11+](https://www.python.org/downloads/)

```bash
source venv/bin/activate  # macOS/Linux

# 升级 pip

# 安装依赖
pip install -r requirements.txt
```

#### 2.2 摄像头采集端
```bash

# 创建虚拟环境

# 安装依赖
pip install -r requirements.txt
```

#### 2.3 Web客户端
```bash
cd web-client

```

- 选择设备和时间范围
- 播放历史录像

## 🔧 常见问题

### Q1: FFmpeg未安装错误
**错误信息**: `[Errno 2] No such file or directory: 'ffmpeg'`

**解决方案**:
```bash
# macOS
brew install ffmpeg

# Ubuntu/Debian
sudo apt install ffmpeg

# 验证安装
ffmpeg -version
```

### Q2: 推流失败 - Broken pipe 错误
**错误信息**: `[Errno 32] Broken pipe`

**原因**: 没有RTSP服务器在运行

**解决方案**:
```bash
# 安装并启动MediaMTX
brew install mediamtx
mediamtx

# 或下载二进制: https://github.com/bluenviron/mediamtx/releases
```

MediaMTX启动后，重新运行摄像头客户端即可。

### Q3: 摄像头无法打开
**解决方案**:
1. 检查摄像头是否被其他程序占用
2. 修改 `camera-client/.env` 中的 `CAMERA_ID`（尝试 0, 1, 2）
3. 确认摄像头权限已授予

### Q4: 服务器启动失败
**解决方案**:
1. 检查端口 8000 是否被占用: `lsof -i :8000`
2. 确认虚拟环境已激活
3. 检查依赖是否完整安装: `pip list`

### Q5: Web界面无法访问
**解决方案**:
1. 确认服务器已启动（http://localhost:8000）
2. 检查端口 5173 是否被占用
3. 清除浏览器缓存后重试

### Q6: 登录失败 (bcrypt错误)
**错误信息**: `password cannot be longer than 72 bytes`

**解决方案**:
已修复！重启服务器即可。如果仍有问题：
```bash
cd server
source venv/bin/activate
pip install --upgrade bcrypt
```

### Q7: 视频无法播放
**解决方案**:
1. 确认MediaMTX已启动（端口8554）
2. 确认摄像头客户端已启动并成功注册
3. 检查浏览器控制台是否有错误
4. 确认 FFmpeg 已正确安装: `ffmpeg -version`

## 📝 开发模式 vs 生产模式

### 开发模式（当前）
- 服务器: `uvicorn main:app --reload`
- Web客户端: `npm run dev`
- 特点: 热重载、详细日志、调试友好

### 生产模式
```bash

cd web-client

# 使用生产配置启动服务器
cd server
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```
