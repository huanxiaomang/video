# 项目启动指令与前置环境清单

> 适用项目：电厂巡检机器人无线视频传输系统
> 目标：给出启动顺序与终端命令（Windows 环境）

---

## 一、前置环境清单（必须完成）

1. **Python 3.8+**
   - 下载：https://www.python.org/downloads/
2. **Node.js 16+**
   - 下载：https://nodejs.org/
3. **MediaMTX 可执行文件**
   - 下载：https://github.com/bluenviron/mediamtx/releases
4. **FFmpeg（可选，用于RTSP备用）**
   - 下载：https://ffmpeg.org/download.html
5. **浏览器**
   - 推荐：Chrome / Edge 最新版

---

## 二、依赖安装（首次运行必做）

### 1. 后端依赖（server）
```bash
cd c:\Users\Administrator\Desktop\video\server
pip install -r requirements.txt
```

### 2. 摄像头端依赖（camera-client）
```bash
cd c:\Users\Administrator\Desktop\video\camera-client
pip install -r requirements.txt
```

### 3. 前端依赖（web-client）
```bash
cd c:\Users\Administrator\Desktop\video\web-client
pnpm install
```

---

## 三、启动顺序（建议按以下顺序开 4 个终端）

### 终端1：启动 MediaMTX
```bash
cd c:\Users\Administrator\Desktop\video
mediamtx.exe mediamtx.yml
```

### 终端2：启动后端服务（FastAPI）
```bash
cd c:\Users\Administrator\Desktop\video\server
python main.py
```

### 终端3：启动前端（Vue）
```bash
cd c:\Users\Administrator\Desktop\video\web-client
pnpm dev
```

### 终端4：启动摄像头推流端
```bash
cd c:\Users\Administrator\Desktop\video\camera-client
python main.py
```

---

## 四、启动后访问地址

- **前端页面**：通常为 `http://localhost:5173`（以终端输出为准）
- **MediaMTX WebRTC**：`http://localhost:8889/{设备ID}/whep`
- **RTSP 备用**：`rtsp://localhost:8554/{设备ID}`

---

## 五、常见问题提示

- 端口冲突：确保 8889 / 8554 / 8000 / 5173 未被占用
- 摄像头打不开：检查 `camera-client/config.py` 中的 CAMERA_ID
- 依赖安装失败：确认 Python/Node 版本正确
