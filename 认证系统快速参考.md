## 🎯 核心概念速查

### Bcrypt（密码加密）

```
作用：安全存储密码
特点：单向加密 + 自动加盐 + 慢速计算
结果：$2b$12$盐值(22字符)哈希值(31字符)
```

**关键函数：**
```python
# 加密
hash_password("admin123") → "$2b$12$..."

# 验证
verify_password("admin123", "$2b$12$...") → True/False
```

---

### JWT Token（身份令牌）

```
作用：验证用户身份
特点：无状态 + 防篡改 + 自动过期
结构：Header.Payload.Signature
```

**关键函数：**
```python
# 生成
create_access_token({"sub": "admin"}) → "eyJhbGci..."

# 验证
jwt.decode("eyJhbGci...", SECRET_KEY) → {"sub": "admin", "exp": 1735689600}
```

**问题引入：**
- "如果没有密码，任何人都能看你的监控视频，安全吗？"

**解决方案：**
- Bcrypt：安全存储密码
- JWT：验证用户身份

---

### 2. Bcrypt演示（8分钟）

**演示1：加密密码**
```python
password = "admin123"
hashed = hash_password(password)
print(hashed)  # $2b$12$KIXxJ3V8h9Zq7X...
```

**关键点：**
- ✅ 密文无法还原成原密码
- ✅ 每次加密结果都不同（盐值不同）
- ✅ 计算慢（4096次迭代），防暴力破解

**演示2：验证密码**
```python
verify_password("admin123", hashed)  # True ✅
verify_password("wrong", hashed)     # False ❌
```

**类比：**
"就像把肉搅成肉馅，无法还原成原来的肉块"

---

### 3. JWT演示（8分钟）

**演示1：生成Token**
```python
token = create_access_token({"sub": "admin"})
print(token)  # eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**在线解码：** https://jwt.io/
- 粘贴Token
- 查看Header、Payload、Signature

**关键点：**
- ✅ Token包含用户信息（用户名）
- ✅ Token有签名保护（无法篡改）
- ✅ Token有过期时间（24小时）

**演示2：验证Token**
```python
payload = jwt.decode(token, SECRET_KEY)
print(payload)  # {"sub": "admin", "exp": 1735689600}
```

**类比：**
"就像一张临时通行证，有效期内可以自由进出，过期就失效"

---

### 4. 完整流程演示（10分钟）

**步骤1：用户注册**
```
输入：username="admin", password="admin123"
处理：hash_password("admin123") → "$2b$12$..."
存储：INSERT INTO users (username, password_hash) VALUES (...)
```

**步骤2：用户登录**
```
输入：username="admin", password="admin123"
验证：verify_password("admin123", "$2b$12$...") → True ✅
生成：create_access_token({"sub": "admin"}) → "eyJhbGci..."
返回：{"access_token": "eyJhbGci...", "token_type": "bearer"}
```

**步骤3：访问资源**
```
请求：GET /api/devices
请求头：Authorization: Bearer eyJhbGci...
验证：jwt.decode("eyJhbGci...", SECRET_KEY) → {"sub": "admin"}
返回：设备列表数据
```

**步骤4：Token过期**
```
24小时后...
请求：GET /api/devices
验证：Token已过期 ❌
返回：401 Unauthorized
前端：跳转到登录页
```

## 🔒 安全性要点

### Bcrypt安全性

| 特性 | 说明 |
|------|------|
| 单向加密 | 无法从密文还原密码 |
| 自动加盐 | 每次加密结果不同 |
| 慢速计算 | 4096次迭代，防暴力破解 |
| 破解时间 | 8位密码需要6900万年 |

### JWT安全性

| 特性 | 说明 |
|------|------|
| 签名验证 | 篡改Token会被立即发现 |
| 过期时间 | 24小时自动失效 |
| 密钥保护 | SECRET_KEY只在服务器 |
| 无状态 | 不存储在数据库 |

## ❓ 常见问题速答

**Q: JWT可以被解码，不是不安全吗？**
A: 可以解码但无法伪造。Token有签名保护，修改会被发现。

**Q: 为什么不用MD5？**
A: MD5太快，容易暴力破解；没有自动加盐，容易被彩虹表攻击。

**Q: Token过期了怎么办？**
A: 用户重新登录，或实现Token刷新机制。

**Q: 密码存在哪里？**
A: 数据库只存密码哈希，不存原密码。

**Q: Token存在哪里？**
A: 前端存在localStorage，每次请求自动添加到请求头。

## 📊 数据对比

### Bcrypt vs 其他加密

| 算法 | 速度 | 加盐 | 安全性 | 推荐 |
|------|------|------|--------|------|
| Bcrypt | 慢 | 自动 | ⭐⭐⭐⭐⭐ | ✅ |
| MD5 | 快 | 手动 | ⭐ | ❌ |
| SHA256 | 快 | 手动 | ⭐⭐ | ❌ |

### JWT vs Session

| 特性 | JWT | Session |
|------|-----|---------|
| 存储 | 客户端 | 服务器 |
| 状态 | 无状态 | 有状态 |
| 扩展性 | 易扩展 | 需共享存储 |
| 失效 | 无法主动失效 | 可立即失效 |
