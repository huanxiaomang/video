# 电厂巡检机器人端到端无线视频传输系统 - 架构与功能设计文档

## 1. 项目概述

### 1.1 项目背景
电厂内环境复杂，存在高温、高压、辐射等危险区域，不适合人工巡检。本系统设计一种端到端的无线视频传输解决方案，通过机器人搭载摄像头进行远程巡检，将实时画面传输到监控端，实现人工故障识别。

### 1.2 技术指标
- **视频分辨率**：≥720P (1280x720)
- **实时性要求**：延迟 ≤1s
- **自适应码率**：带宽受限时自动调节，减少丢帧和卡顿
- **开发语言**：Python (服务端) + Vue3 + TypeScript (前端)
- **传输协议**：WebRTC / RTSP / HLS (根据场景选择)

### 1.3 系统组成
- **视频采集端**：Python服务，获取摄像头数据并编码压缩
- **传输服务器**：Python后端，处理视频流转发、编码、打包
- **监控客户端**：Vue3 + TypeScript Web应用，实时播放视频流

## 4. 视频传输协议选型

### 4.1 协议对比

| 协议 | 延迟 | 兼容性 | 复杂度 | 适用场景 |
|------|------|--------|--------|----------|
| **WebRTC** | 100-500ms | 浏览器原生支持 | 高 | 实时监控（推荐） |
| **RTSP** | 1-3s | 需要插件/转码 | 中 | 设备推流 |
| **HLS** | 5-10s | 广泛支持 | 低 | 录像回放 |
| **RTMP** | 2-5s | 需要Flash/转码 | 中 | 传统直播 |

### 4.2 推荐方案

**采集端 → 服务器**：RTSP
- 摄像头使用FFmpeg推送RTSP流到服务器
- 稳定可靠，易于实现

**服务器 → 客户端**：WebRTC
- 低延迟，满足实时性要求
- 浏览器原生支持，无需插件
- 支持自适应码率

**录像回放**：HLS
- 兼容性好，易于实现
- 支持分段加载，节省带宽

## 5. 码率自适应策略

### 5.1 网络状况检测

```python
# 网络质量评估指标
class NetworkQuality:
    bandwidth: int        # 可用带宽 (kbps)
    rtt: int             # 往返时延 (ms)
    packet_loss: float   # 丢包率 (0-1)
    jitter: int          # 抖动 (ms)

    def get_quality_level(self) -> str:
        """返回网络质量等级：excellent/good/fair/poor"""
        if self.bandwidth > 3000 and self.packet_loss < 0.01:
            return 'excellent'
        elif self.bandwidth > 1500 and self.packet_loss < 0.05:
            return 'good'
        elif self.bandwidth > 600 and self.packet_loss < 0.1:
            return 'fair'
        else:
            return 'poor'
```

### 5.2 码率调整策略

```python
# 码率档位配置
BITRATE_PROFILES = {
    'high': {
        'resolution': (1920, 1080),
        'fps': 30,
        'bitrate': 4000,  # kbps
        'min_bandwidth': 3000,
    },
    'medium': {
        'resolution': (1280, 720),
        'fps': 25,
        'bitrate': 2000,
        'min_bandwidth': 1500,
    },
    'low': {
        'resolution': (854, 480),
        'fps': 15,
        'bitrate': 800,
        'min_bandwidth': 600,
    },
    'ultra_low': {
        'resolution': (640, 360),
        'fps': 10,
        'bitrate': 400,
        'min_bandwidth': 300,
    }
}

# 自适应算法
def adjust_bitrate(network_quality: NetworkQuality) -> dict:
    """根据网络质量调整码率"""
    bandwidth = network_quality.bandwidth
    packet_loss = network_quality.packet_loss

    if bandwidth > 3000 and packet_loss < 0.01:
        return BITRATE_PROFILES['high']
    elif bandwidth > 1500 and packet_loss < 0.05:
        return BITRATE_PROFILES['medium']
    elif bandwidth > 600 and packet_loss < 0.1:
        return BITRATE_PROFILES['low']
    else:
        return BITRATE_PROFILES['ultra_low']
```

### 5.3 缓冲区管理

```python
class BufferManager:
    """视频缓冲区管理"""
    def __init__(self, target_buffer_ms: int = 500):
        self.target_buffer = target_buffer_ms  # 目标缓冲时长
        self.current_buffer = 0
        self.max_buffer = 2000  # 最大缓冲2秒

    def should_drop_frame(self) -> bool:
        """判断是否应该丢帧"""
        return self.current_buffer > self.max_buffer

    def should_request_keyframe(self) -> bool:
        """判断是否应该请求关键帧"""
        return self.current_buffer < 100  # 缓冲不足100ms
```

```python
import cv2
import numpy as np

class CameraCapture:
    def __init__(self, camera_id: int = 0, resolution: tuple = (1280, 720), fps: int = 25):
        self.camera_id = camera_id
        self.resolution = resolution
        self.fps = fps
        self.cap = None

    def start(self):
        """启动摄像头"""
        self.cap = cv2.VideoCapture(self.camera_id)
        self.cap.set(cv2.CAP_PROP_FRAME_WIDTH, self.resolution[0])
        self.cap.set(cv2.CAP_PROP_FRAME_HEIGHT, self.resolution[1])
        self.cap.set(cv2.CAP_PROP_FPS, self.fps)

    def read_frame(self) -> np.ndarray:
        """读取一帧"""
        ret, frame = self.cap.read()
        if not ret:
            raise Exception("Failed to read frame")
        return frame

    def stop(self):
        """停止摄像头"""
        if self.cap:
            self.cap.release()
```

#### 7.1.2 H.264编码 (FFmpeg)

```python
import subprocess
import threading
import queue

class H264Encoder:
    def __init__(self, width: int, height: int, fps: int, bitrate: int):
        self.width = width
        self.height = height
        self.fps = fps
        self.bitrate = bitrate
        self.process = None

    def start(self, output_url: str):
        """启动编码器"""
        cmd = [
            'ffmpeg',
            '-f', 'rawvideo',
            '-pix_fmt', 'bgr24',
            '-s', f'{self.width}x{self.height}',
            '-r', str(self.fps),
            '-i', '-',  # 从stdin读取
            '-c:v', 'libx264',
            '-preset', 'ultrafast',
            '-tune', 'zerolatency',
            '-b:v', f'{self.bitrate}k',
            '-maxrate', f'{self.bitrate}k',
            '-bufsize', f'{self.bitrate * 2}k',
            '-g', str(self.fps * 2),  # GOP大小
            '-f', 'rtsp',
            output_url
        ]

        self.process = subprocess.Popen(
            cmd,
            stdin=subprocess.PIPE,
            stdout=subprocess.PIPE,
            stderr=subprocess.PIPE
        )

    def encode_frame(self, frame: np.ndarray):
        """编码一帧"""
        if self.process and self.process.stdin:
            self.process.stdin.write(frame.tobytes())

    def stop(self):
        """停止编码器"""
        if self.process:
            self.process.stdin.close()
            self.process.wait()
```

#### 7.1.3 设备注册与心跳

```python
import requests
import time
import threading
from typing import Dict

class DeviceManager:
    def __init__(self, server_url: str, device_info: Dict):
        self.server_url = server_url
        self.device_info = device_info
        self.device_id = None
        self.heartbeat_thread = None
        self.running = False

    def register(self) -> str:
        """注册设备"""
        response = requests.post(
            f'{self.server_url}/api/devices/register',
            json=self.device_info
        )
        response.raise_for_status()
        self.device_id = response.json()['device_id']
        return self.device_id

    def start_heartbeat(self, interval: int = 10):
        """启动心跳"""
        self.running = True
        self.heartbeat_thread = threading.Thread(target=self._heartbeat_loop, args=(interval,))
        self.heartbeat_thread.start()

    def _heartbeat_loop(self, interval: int):
        """心跳循环"""
        while self.running:
            try:
                requests.post(
                    f'{self.server_url}/api/devices/{self.device_id}/heartbeat',
                    json={'timestamp': time.time()}
                )
            except Exception as e:
                print(f"Heartbeat failed: {e}")
            time.sleep(interval)

    def stop_heartbeat(self):
        """停止心跳"""
        self.running = False
        if self.heartbeat_thread:
            self.heartbeat_thread.join()
```

#### 7.2.2 WebSocket流处理

```python
from fastapi import WebSocket, WebSocketDisconnect
from typing import Dict, Set
import asyncio

class StreamConnectionManager:
    def __init__(self):
        self.active_connections: Dict[str, Set[WebSocket]] = {}

    async def connect(self, device_id: str, websocket: WebSocket):
        """连接WebSocket"""
        await websocket.accept()
        if device_id not in self.active_connections:
            self.active_connections[device_id] = set()
        self.active_connections[device_id].add(websocket)

    def disconnect(self, device_id: str, websocket: WebSocket):
        """断开WebSocket"""
        if device_id in self.active_connections:
            self.active_connections[device_id].discard(websocket)
            if not self.active_connections[device_id]:
                del self.active_connections[device_id]

    async def broadcast(self, device_id: str, message: dict):
        """广播消息"""
        if device_id in self.active_connections:
            disconnected = set()
            for connection in self.active_connections[device_id]:
                try:
                    await connection.send_json(message)
                except:
                    disconnected.add(connection)
            # 清理断开的连接
            for conn in disconnected:
                self.disconnect(device_id, conn)

manager = StreamConnectionManager()

@app.websocket("/ws/stream/{device_id}")
async def websocket_stream(websocket: WebSocket, device_id: str):

    try:
        while True:
            data = await websocket.receive_json()
            # 处理接收到的消息
            await handle_stream_message(device_id, data)
    except WebSocketDisconnect:
        manager.disconnect(device_id, websocket)
```

```typescript
          videoRef.value.srcObject = event.streams[0];
        }
      };

      await peerConnection.value.setLocalDescription(offer);

        new RTCSessionDescription({ type: 'answer', sdp: answer })
      );
    };

      initWebRTC();
    });

    onUnmounted(() => {
      if (peerConnection.value) {
        peerConnection.value.close();
      }
    });

    return { videoRef };
  }
};
```

#### 7.3.2 WebSocket状态同步

```typescript
// utils/websocket.ts
import { ref } from 'vue';
import type { WSMessage, WSMessageType } from '@/types';

class WebSocketManager {
  private ws: WebSocket | null = null;
  private reconnectTimer: number | null = null;
  private listeners: Map<WSMessageType, Set<Function>> = new Map();

  public connected = ref(false);

  connect(url: string) {
    this.ws = new WebSocket(url);

    this.ws.onopen = () => {
      this.connected.value = true;
      console.log('WebSocket connected');
    };

    this.ws.onmessage = (event) => {
      const message: WSMessage = JSON.parse(event.data);
      this.handleMessage(message);
    };

    this.ws.onclose = () => {
      this.connected.value = false;
      console.log('WebSocket disconnected');
      this.reconnect(url);
    };

    this.ws.onerror = (error) => {
      console.error('WebSocket error:', error);
    };
  }

  disconnect() {
    if (this.reconnectTimer) {
      clearTimeout(this.reconnectTimer);
    }
    if (this.ws) {
      this.ws.close();
      this.ws = null;
    }
  }

  send(message: WSMessage) {
    if (this.ws && this.ws.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify(message));
    }
  }

  on(type: WSMessageType, callback: Function) {
    if (!this.listeners.has(type)) {
      this.listeners.set(type, new Set());
    }
    this.listeners.get(type)!.add(callback);
  }

  off(type: WSMessageType, callback: Function) {
    if (this.listeners.has(type)) {
      this.listeners.get(type)!.delete(callback);
    }
  }

  private handleMessage(message: WSMessage) {
    if (this.listeners.has(message.type)) {
      this.listeners.get(message.type)!.forEach(callback => {
        callback(message.data);
      });
    }
  }

  private reconnect(url: string) {
    this.reconnectTimer = window.setTimeout(() => {
      console.log('Reconnecting WebSocket...');
      this.connect(url);
    }, 3000);
  }
}

export const wsManager = new WebSocketManager();
```

### 8.2 技术难点与解决方案

#### 8.2.1 低延迟视频传输
**难点**：实现≤1s的端到端延迟

**解决方案**：
1. 采用WebRTC协议，利用UDP传输减少延迟
2. 使用H.264编码的ultrafast预设，降低编码延迟
3. 减小GOP大小（2秒），加快关键帧生成
4. 优化缓冲区大小，平衡延迟与流畅性
5. 使用硬件编码（如果可用）

#### 8.2.2 自适应码率控制
**难点**：在带宽波动时保持流畅播放

**解决方案**：
1. 实时监控网络质量（带宽、RTT、丢包率）
2. 预设多档码率配置（high/medium/low/ultra_low）
3. 根据网络状况动态切换码率
4. 实现平滑过渡，避免突变
5. 使用缓冲区管理策略，防止卡顿

#### 8.2.3 多路并发流处理
**难点**：服务器同时处理多个视频流

**解决方案**：
1. 使用异步I/O（FastAPI + asyncio）
2. 采用多进程/多线程架构
3. 使用专业流媒体服务器（MediaMTX）
4. 实现流复用，减少重复转码
5. 优化资源分配，限制单流资源占用

#### 8.2.4 WebRTC NAT穿透
**难点**：不同网络环境下的P2P连接

**解决方案**：
1. 配置STUN服务器（公网IP发现）
2. 配置TURN服务器（中继转发）
3. 实现ICE候选收集与交换
4. 提供降级方案（WebSocket传输）
